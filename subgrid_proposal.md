# サブグリッド（解像度階層）システム設計案 - 解像度調整版

## 1. 目的

サーバー（令制国）の物理面積に関わらず、タクティカルレイヤーにおける「総マス数」を一定の範囲（400〜3,500マス程度）に収めることで、戦略密度とプレイヤー1人あたりの占有率を正規化します。

## 2. カテゴリ別・正規化分割ルール

親タイル（Parent Hex）の数に応じて、1枚あたりの分割解像度を調整します。

| カテゴリ                 |   代表的な親タイル数   | 推奨分割（1枚あたり） | サーバー内 総マス数（推定） | プレイヤー数 | 1人あたり占有想定 |
| :----------------------- | :--------------------: | :-------------------: | :-------------------------: | :----------: | :---------------: |
| **SS: 蝦夷(道南)** | 約750 (北緯42.8度以下) |    **2 x 2**    |          約 3,000          |    100人    |        30        |
| **S: 超巨大**      |        100〜140        |    **5 x 5**    |       2,500 〜 3,500       |    100人    |     25 〜 35     |
| **A: 有力**        |         40〜80         |    **6 x 6**    |       1,440 〜 2,880       |     50人     |     28 〜 57     |
| **B: 小規模**      |         10〜30         |    **8 x 8**    |        640 〜 1,920        |     30人     |     20 〜 60     |
| **C: 隔離**        |          4〜8          |   **10 x 10**   |         400 〜 800         |     10人     |     40 〜 80     |

### 特殊事例：蝦夷地（SS）

- **道南フォーカス**: 蝦夷地全域（2,763枚）をカバーするとデータが膨大（2.5万マス〜）になるため、**北緯42.8度以下（道南エリア）**のみをタクティカル対象（2x2分割）とし、約3,000マスの「巨大国と同等の戦場」として定義します。

## 3. サブグリッドJSON構造 (蝦夷_sub_001.json)

親タイル1枚ごとに以下の形式のメタデータを生成します。

```json
{
  "server_id": "蝦夷_donan_069",
  "parent_polygon_id": "蝦夷_arcgis_123",
  "subgrid_resolution": "2x2",
  "category": "SS",
  "cells": [ ... ]
}
```

## 4. 保存・管理ルール

- フォルダ: `sengoku/data/{令制国名}/`
- ファイル: `{令制国名}_sub_{seq}.json`

## 5. 次のステップ

1. **正規化後の解像度で仮データを再生成**。
2. 標高データサンプリングロジックの検討。
